# The Ultimate Hand-Holding Guide: Building a 3D Storytelling E-Commerce Website

This guide will walk you through **every single step** required to build a high-end, award-winning 3D website. We will use the **React ecosystem** because it makes handling 3D scenes (via React Three Fiber) much easier than vanilla JavaScript.

---

## 🏁 Phase 1: Preparation & Setup (Don't Skip!)

Before you write a single line of code, you need the right tools.

### 1. Install Node.js
You need Node.js to run the development server.
-   **Download**: Go to [nodejs.org](https://nodejs.org/) and download the **LTS (Long Term Support)** version.
-   **Verify**: Open your terminal (Command Prompt or PowerShell) and type:
    ```bash
    node -v
    npm -v
    ```
    If you see version numbers, you're good.

### 2. Create Your Project
We will use **Vite** because it's lightning fast.
1.  Open your terminal in the folder where you want your project.
2.  Run this command:
    ```bash
    npm create vite@latest my-3d-shop -- --template react
    ```
3.  Go into your new project:
    ```bash
    cd my-3d-shop
    ```
4.  Install the basic dependencies:
    ```bash
    npm install
    ```

### 3. Install the "Magic" Libraries
These are the specific tools that obtain the 3D look and feel. Run this command:

```bash
npm install three @types/three @react-three/fiber @react-three/drei gsap @gsap/react lenis perfect-scrollbar framer-motion
```

**Why these?**
*   **Three.js**: The 3D engine.
*   **@react-three/fiber (R3F)**: Lets you write 3D objects like HTML components (`<mesh />`).
*   **@react-three/drei**: Gives you pre-made helpers like `<OrbitControls>`, `<Environment>`, and model loaders.
*   **GSAP**: The best animation library for scrolling.
*   **Lenis**: Makes scrolling smooth, which is **crucial** for 3D experiences.

## 🎨 Phase 2: Finding & Preparing Your 3D Model

This is where most beginners fail. You cannot just download a random 3D file and expect it to work.

### 1. Find a Model
For an e-commerce store (e.g., a shoe, a chair, a bottle), you need a high-quality but **low-poly** model.
-   **Sources**: [Sketchfab](https://sketchfab.com/), [PMNDRS Market](https://market.pmnd.rs/), or hire a 3D artist.
-   **Format**: Look for **.gltf** or **.glb**.

### 2. Compress the Model (CRITICAL)
A raw 3D scan can be 50MB. Ideally, your model should be **under 5MB**.
-   **Tool**: Go to [gltf.report](https://gltf.report/).
-   **Step**: Drag your `.glb` file there.
-   **Step**: Click **Script** -> **Compress** -> choose **Draco**.
-   **Step**: Export the file. It will be much smaller.

### 3. Convert to React Component
We don't want to load specific parts of the model by hand. We want a React component.
-   **Tool**: Go to [gltf.pmnd.rs](https://gltf.pmnd.rs/).
-   **Step**: Drag your compressed `.glb` file there.
-   **Step**: Copy the code it generates.
-   **Action**: Create a file in `src/components/Model.jsx` and paste that code. It will look like this:

```jsx
/*
Auto-generated by: https://github.com/pmndrs/gltfjsx
*/
import React, { useRef } from 'react'
import { useGLTF } from '@react-three/drei'

export function Model(props) {
  const { nodes, materials } = useGLTF('/shoe-draco.glb')
  return (
    <group {...props} dispose={null}>
      <mesh geometry={nodes.shoe.geometry} material={materials.laces} />
    </group>
  )
}
useGLTF.preload('/shoe-draco.glb') // This preloads it!
```
-   **Action**: Put your `.glb` file in the `public/` folder of your project.

## 💻 Phase 3: Building the Scene

Now, let's render it.

### 1. Set up the Canvas
In your `App.jsx`, replace everything with this:

```jsx
import { Canvas } from '@react-three/fiber'
import { Environment, OrbitControls } from '@react-three/drei'
import { Model } from './components/Model' // Import your model

function App() {
  return (
    <div className="App" style={{ height: '100vh', width: '100vw' }}>
        <Canvas camera={{ position: [0, 0, 5], fov: 45 }}>
            <color attach="background" args={['#1a1a1a']} /> {/* Dark background */}
            
            {/* Lighting */}
            <ambientLight intensity={0.5} />
            <spotLight position={[10, 10, 10]} angle={0.15} penumbra={1} />
            <Environment preset="city" /> {/* Realistic reflections */}
            
            {/* The 3D Object */}
            <Model scale={1.5} />
            
            <OrbitControls enableZoom={false} />
        </Canvas>
    </div>
  )
}
export default App;
```

**Checkpoint**: Run `npm run dev`. You should see your 3D model floating in the center!

## ✨ Phase 4: The "Storytelling" (Scroll Animation)

This is the secret sauce. We want the model to rotate and move as the user scrolls.

### 1. Overlay HTML Content
We need a long page to scroll through. Add this AFTER the `<Canvas>` in `App.jsx`:

```jsx
<div className="overlay" style={{ position: 'absolute', top: 0, left: 0, width: '100%', pointerEvents: 'none' }}>
    <section style={{ height: '100vh', display: 'flex', alignItems: 'center', justifyContent: 'center' }}>
        <h1 style={{ color: 'white', fontSize: '5rem' }}>The Future of Sneakers</h1>
    </section>
    <section style={{ height: '100vh', display: 'flex', alignItems: 'center', justifyContent: 'flex-end', paddingRight: '10%' }}>
        <h2 style={{ color: 'white', fontSize: '3rem' }}>Lightweight Design</h2>
    </section>
    <section style={{ height: '100vh', display: 'flex', alignItems: 'center', justifyContent: 'flex-start', paddingLeft: '10%' }}>
        <h2 style={{ color: 'white', fontSize: '3rem' }}>Sustainable Materials</h2>
    </section>
</div>
```

### 2. Animate Only on Scroll
Go back to your `Model.jsx`. We need to use **GSAP**.

```jsx
import React, { useRef, useLayoutEffect } from 'react'
import { useGLTF, useScroll } from '@react-three/drei'
import { useFrame } from '@react-three/fiber'
import gsap from 'gsap'
import { ScrollTrigger } from 'gsap/ScrollTrigger'

gsap.registerPlugin(ScrollTrigger)

export function Model(props) {
  const { nodes, materials } = useGLTF('/shoe-draco.glb')
  const ref = useRef()
  const tl = useRef()

  useLayoutEffect(() => {
    // Determine the scroll timeline
    tl.current = gsap.timeline({
        scrollTrigger: {
            trigger: "body", // The entire page is the trigger
            start: "top top",
            end: "bottom bottom",
            scrub: 1, // Smoothly scrub the animation
        }
    })

    // Animation 1: Rotate 360 degrees
    tl.current.to(ref.current.rotation, { y: Math.PI * 2, duration: 2 })
    
    // Animation 2: Move slightly to the left
    tl.current.to(ref.current.position, { x: -2 }, 0) // The '0' plays it at the start

    // Animation 3 (Scene 2): Move to the right for the second text
    tl.current.to(ref.current.position, { x: 2, y: 1 }, 2) // Plays after 2 seconds (relative to timeline)
  }, [])

  return (
    <group {...props} dispose={null} ref={ref}>
      <mesh geometry={nodes.shoe.geometry} material={materials.laces} />
    </group>
  )
}
```

## 🚀 Phase 5: Optimization Checklist (Before Launch)

1.  **Loader**: Add `<Suspense fallback={<Loader />}>` around your `<Model />` inside the Canvas. Use the `Loader` from `@react-three/drei`.
2.  **Mobile Support**: Check the site on your phone.
    -   Use `gsap.matchMedia()` to run different animations for mobile (move objects up/down instead of left/right).
3.  **Shadows**: Shadows are expensive. Only enable `castShadow` on your main object and `receiveShadow` on the floor if you really need them.
4.  **Dispose**: Ensure you aren't creating new geometries inside the render loop (i.e., don't do `new THREE.BoxGeometry()` inside `useFrame`).
